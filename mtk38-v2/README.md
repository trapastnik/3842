# mtk38-v2 — версия 2 МТК 38 (арт-объект «Ленин на языках мира»)

Отдельное пространство v2, параллельное v1 (`mtk38-globe/`, `mtk38-map/`, …). v1 остаётся
нетронутым. v2 деплоится **по отдельному пути** (напр. `…/mtk38-v2/`).

Рефакторинг «порядок и красота» (2026-07-07): `app.html`+`studio.html` сведены в **один**
арт-объект; настройки — **единая панель, выезжающая справа**; структура разделена на
**фронт** (что видит посетитель) и **бэк** (данные, генераторы, приёмка).

## Структура

```
mtk38-v2/
├── index.html            ← лендинг-хаб: фронт-композиции + бэк-инструменты
├── front/                ← ФРОНТ · визуальные композиции (посетитель)
│   ├── studio.html       ← ★ АРТ-ОБЪЕКТ: единый WebGL-движок, 7 композиций (глобус/облако/
│   │                       мандала/дождь/лента/стена/карта), слой цитат (RU+EN), тап→карточка
│   │                       языка (Р2). Данные — канон в рантайме. Диплинк ?layout=globe&q=1
│   └── map.html          ← Canvas-2D карта мира, гео-привязка точек, идеологические маршруты
├── back/                 ← БЭК · данные / генераторы / приёмка
│   ├── editor.html       ← редактор всех полей data/mtk38.json + вкладка «Приёмка» (сверка с листом)
│   ├── validate.html     ← редирект на editor.html#review (валидатор объединён с редактором)
│   └── analysis.html     ← репрезентативность, охват, предложения
├── shared/               ← переиспользуемые модули (подключаются на всех страницах)
│   ├── panel.css/.js     ← ★ ЕДИНАЯ панель настроек (window.Panel) — выезжает справа, оверлей,
│   │                       не сдвигает контент. groups/toggles/segments, копировать/сброс, hotkey H
│   └── data.js           ← рантайм-загрузчик канона (window.MTK38Data.load) + встроенный fallback
├── fonts/
│   ├── faces.css         ← единый @font-face (бренд + 29 письменностей Noto) — один <link> вместо дублей
│   └── noto/             ← subset-Noto (29 письменностей × 400/700), SIL OFL
└── vendor/three/         ← Three.js r137 UMD + bloom-addons (вендорено, без CDN)
```

## Ключевые решения рефакторинга

- **Один арт-объект.** `front/studio.html` = ширина `app.html` (7 композиций в одном движке) +
  качество `studio.html` (текстуры слов по `actualBoundingBox` — сложные письменности не режутся;
  виньетка/зерно; карточка Р2; цитаты RU+EN с 5 режимами размещения). Старые `app.html`/`studio.html`
  удалены (история — в git + тегах `archive/mtk38-*`).
- **Единая панель.** Оба фронт-варианта (арт-объект и карта) используют один компонент
  `shared/panel.js` (+`panel.css`, автоподключается). Прежние 4 разнородные панели (app `#panel`,
  map `#panel`, `tuner.js`, v3 `createTuner`) на фронте v2 сведены к одной. Панель — оверлей справа
  с язычком-ручкой; не смещает сцену.
- **Фронт читает бэк в рантайме.** `data/mtk38.json` (**53 языка**) + `data/mtk38-quotes.json`
  (13 цитат) — единый источник. `shared/data.js` тянет их по http; при `file://` (где `fetch`
  недоступен) отдаёт встроенный слепок. Правка канона подхватывается фронтом без пересборки.
- **Движки v2/v3 не объединялись** (осознанно). v2 = Three r137 UMD, работает и по `file://`;
  v3 = Three r184 WebGPU (ES-модули, только http). Слияние под демо-дедлайн — высокий риск при
  малой пользе; каждый движок валиден в своей нише. Возможное объединение — отдельная задача.

## Генераторы (mtk38-handoff)

Ранее все страницы генерировались `build_*.py` из `data/mtk38.json`. После рефакторинга фронт
(`front/studio.html`) — **рукописный** и читает канон в рантайме, поэтому `build_app.py` и
`build_studio.py` **устарели**. `build_editor.py`/`build_analysis.py`/`build_map.py` ещё
соответствуют своим файлам, но их вывод теперь лежит в `front/`/`back/` — при пересборке
поправить пути. Генераторы оставлены как провенанс; не запускать вслепую.

## Запуск

Фронт с рантайм-данными требует **http** (не `file://` — иначе fallback-слепок):
из корня репозитория поднять статику и открыть `mtk38-v2/index.html`. Панель — клавиша **H**.
Прод = офлайн-киоск (свой локальный сервер), VPS = демо.

## Координатору (FYI, не действие)

- `mtk38-v2/` — **не вариант v1-хаба**. Не добавляй в `window.HUB_VARIANTS` для `/mtk38/`.
  Это самостоятельный путь со своим лендингом.
- Ссылку на v2 с главной (`index.html`) добавлять **по желанию пользователя**, не автоматически.
- Деплой v2 = тот же `git pull` на VPS, что и всё остальное (путь раздаётся существующим nginx).
- **Изменились пути** после рефакторинга: `mtk38-v2/validate.html` → `mtk38-v2/back/validate.html`,
  `app.html`/`studio.html` → `mtk38-v2/front/studio.html`. `COORDINATION.md` (стр. ~281/321) ссылается
  на старый путь `mtk38-v2/validate.html` — поправить при сведении (я в COORDINATION.md не пишу).
- **Единый хаб** — теперь ОДИН: `mtk38-v2/index.html` (фронт-композиции + движок v3 WebGPU + бэк-инструменты).
  Прежний `mtk38-v3/demo.html` больше не отдельный лендинг — это **редирект** на `index.html`
  (старый вход на 8097 не ломается). Точка входа для показа/деплоя = `mtk38-v2/index.html`.
