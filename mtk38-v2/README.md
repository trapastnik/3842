# mtk38-v2 — версия 2 МТК 38 (отдельный путь-деплой)

Это **отдельное пространство v2**, параллельное v1 (`mtk38-globe/`, `mtk38-map/`, …).
v1 остаётся нетронутым и живым. v2 деплоится **по отдельному пути** на текущем домене,
напр. `38-42.mtk.ostrov-vezeniya.ru/mtk38-v2/`.

Решение пользователя (2026-06-10): v2 через единый сервер + пайплайн (merge → 8092 → VPS),
**отдельный адрес = отдельный путь, без infra-возни**. Координаторскую infra-зону
(nginx/docker, поддомены) не трогаем.

## Содержимое

```
mtk38-v2/
├── index.html       ← лендинг: арт-объект главным + 7 кликабельных композиций + инструменты
├── app.html         ← ★ АРТ-ОБЪЕКТ: единый WebGL-движок, 7 композиций (глобус/облако/мандала/
│                       дождь/лента/стена/карта), тюнящаяся панель, тап→карточка языка (Р2),
│                       слой цитат. Диплинк ?layout=globe&q=1. Генерится build_app.py
├── validate.html    ← валидатор написаний для заказчика (52 языка). build_validate.py
├── analysis.html    ← аналитика репрезентативности + предложения. build_analysis.py
├── render-test.html ← сэндбокс рендера глобуса (крутилки). build_render.py
├── quotes-test.html ← сэндбокс размещения/рендера цитат. build_quotes_viz.py
├── vendor/three/    ← Three.js r137 UMD + bloom-addons (вендорено, без CDN)
└── fonts/noto/      ← subset-Noto, 29 письменностей (~140 КБ, 400+700) для рендера не-латиницы
```

Движок (app.html) — на наработанных в этом цикле технологиях: WebGL/Three.js + UnrealBloom +
туман + ACES, спрайты-написания реальными шрифтами (20 Kopeek / Arial Unicode MS / Noto),
общая панель параметров (всё настраивается), карточки языка (Р2), цитаты. Все генераторы —
в `../mtk38-handoff/`, источник данных — `data/mtk38.json` + `data/mtk38-quotes.json`.
Шрифты не-латиницы для **киоска** (полные Noto) — отдельная задача координатора
(`assets/shared/fonts/noto/`); тут лежат subset'ы для рендера написаний.

## Координатору (FYI, не действие)

- `mtk38-v2/` — **не вариант v1-хаба**. Не добавляй в `window.HUB_VARIANTS` для `/mtk38/`.
  Это самостоятельный путь со своим лендингом.
- Ссылку на v2 с главной (`index.html`) добавлять **по желанию пользователя**, не автоматически.
- Деплой v2 = тот же `git pull` на VPS, что и всё остальное (путь раздаётся существующим nginx).
