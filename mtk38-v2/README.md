# mtk38-v2 — версия 2 МТК 38 (отдельный путь-деплой)

Это **отдельное пространство v2**, параллельное v1 (`mtk38-globe/`, `mtk38-map/`, …).
v1 остаётся нетронутым и живым. v2 деплоится **по отдельному пути** на текущем домене,
напр. `38-42.mtk.ostrov-vezeniya.ru/mtk38-v2/`.

Решение пользователя (2026-06-10): v2 через единый сервер + пайплайн (merge → 8092 → VPS),
**отдельный адрес = отдельный путь, без infra-возни**. Координаторскую infra-зону
(nginx/docker, поддомены) не трогаем.

## Содержимое

```
mtk38-v2/
├── index.html      ← v2-лендинг (самодостаточный, бренд-шрифты из ../mtk38-globe/fonts/)
├── validate.html   ← инструмент валидации написаний для заказчика (52 языка, генерится)
├── globe/ map/ …   ← v2-прототипы (Ф2, в работе)
```

`validate.html` генерируется из `data/mtk38.json` скриптом `../mtk38-handoff/build_validate.py`
(единый источник истины — JSON; правки данных → перегенерация). Лендинг и прототипы — вручную.

## Координатору (FYI, не действие)

- `mtk38-v2/` — **не вариант v1-хаба**. Не добавляй в `window.HUB_VARIANTS` для `/mtk38/`.
  Это самостоятельный путь со своим лендингом.
- Ссылку на v2 с главной (`index.html`) добавлять **по желанию пользователя**, не автоматически.
- Деплой v2 = тот же `git pull` на VPS, что и всё остальное (путь раздаётся существующим nginx).
