# SETUP.md — Подключение новой Claude-сессии к проекту

Этот файл — onboarding для новой сессии в любом из worktree. Прочитай его целиком до начала работы.

## Шаг 1. Определи свой МТК

В этом проекте 5 мультимедийных комплексов (МТК). Каждый — отдельный арт-объект с интерактивным экраном:

| № | Тема | Что это |
|---|---|---|
| **МТК 38** | «Ленин на языках мира» | Композиция из слова «Ленин» на 42 языках/письменностях |
| **МТК 39** | «Имени Ленина» | Объекты, носящие имя В.И. Ленина: вузы, заводы, ГЭС, шахты, библиотеки |
| **МТК 40** | «Библиотека Ленина» | Литература о Ленине и его собственные работы |
| **МТК 41** | «Скульптуры» | Памятники В.И. Ленину |
| **МТК 42** | «Осмысление наследия» | Современный взгляд на личность, цитаты, исследователи |

**Спроси пользователя**: «Над каким МТК работаем в этой сессии?» (если ещё не сказано).

Дальше везде `XX` = твой номер МТК.

## Шаг 2. Файловые границы

### Твоё (правишь свободно)

- `mtkXX-<имя-варианта>/` — каждый прототип в своей подпапке
- `data/mtkXX.json` — контент твоего МТК (если есть)
- Секция «МТК XX» в [`README.md`](README.md)
- Карточки в секции «МТК XX» в корневом [`index.html`](index.html)

### Чужое (НЕ трогай)

- `mtkYY-*/` и `data/mtkYY.json` для любого `YY ≠ XX`
- Секции других МТК в README и index.html

### Общее, только чтение

- `data/ne_110m_countries.geojson` — границы стран Natural Earth (110m). Используется для географических вариантов.
- `mtk38-globe/fonts/` — шрифты бренда (Nolde, 21 Cent, 20 Kopeek). Подключай через `../mtk38-globe/fonts/...` относительно своего варианта. В будущем переедут в `assets/fonts/` (этап shell в [PLAN.md](PLAN.md)).
- [`PLAN.md`](PLAN.md) — архитектура. Читать, чтобы понимать общую картину. Менять только после обсуждения через [`COORDINATION.md`](COORDINATION.md).

### Append-only

- [`COORDINATION.md`](COORDINATION.md) — туда пишут все при изменении правил или фиксации событий (например, merge). Добавляй секции, не перезаписывай существующие без согласования.

## Шаг 3. Git

Ты работаешь в **git worktree** — отдельной рабочей копии под свою feature-ветку. Ветка уже создана, она называется примерно `claude/<имя-worktree>`.

### Базовые правила

- **Коммить в свою ветку**, не в main напрямую.
- Сообщения коммитов на русском или английском — стиль императивный («Add ...», «Fix ...», «Remove ...»).
- Не запускай `git push` без явной просьбы пользователя.
- Не делай `git rebase -i`, `git reset --hard`, `--force` — спроси перед любой деструктивной операцией.

### Типичный цикл

```bash
# Подтянуть свежий main
git fetch
git merge main

# Сделать работу, потом
git add mtkXX-<вариант>/   data/mtkXX.json   README.md   index.html
git commit -m "Add mtkXX-<вариант> variant"
```

### Слияние в main

Когда вариант готов:

1. Сообщи пользователю, что хочешь смержить.
2. Если пользователь даёт добро — либо merge через main, либо они делают PR.

Конфликтов с другими сессиями быть не должно: каждый правит только свои файлы. Если конфликтует README/index.html — это значит чужой регион случайно задет; откатить.

## Шаг 4. HTTP-сервер

**Один общий сервер на порту 8092** из корня репозитория:

```bash
cd <repo-root>     # директория BMK/38-42
python3 -m http.server 8092
```

После запуска посадочная — http://127.0.0.1:8092/. Там видно все МТК всех сессий.

Если уже запущен (порт занят) и серверит **не из корня репо** — убей и перезапусти из корня:

```bash
lsof -ti:8092 | xargs kill -9
cd <repo-root>
python3 -m http.server 8092
```

## Шаг 5. Соглашения по коду варианта

Каждый прототип — самостоятельная папка `mtkXX-<имя>/`, содержит:

- `index.html` — точка входа, минимальный wrapper с canvas и script
- `styles.css` — брендовая палитра, фон, шрифты (`@font-face` ссылается на `../mtk38-globe/fonts/...`)
- `<имя>.js` — основной скрипт варианта

Базовая структура — см. любой существующий `mtk38-*` как образец.

### Брендовые токены (используй везде)

```css
:root {
  --paper:    #F7F9EF;
  --brass:    #D2B773;
  --red:      #A02128;
  --graphite: #435059;
  --window:   #9DA3A6;
}
```

Шрифты: `Nolde` (display, латиница / кириллица / греческий), `21 Cent` (body), `20 Kopeek` (mono / технические подписи).

### Тач-интерфейс (важно для киоска)

- Никаких hover-состояний — это сенсорный экран
- Визуальный фидбек только при активном касании / перетаскивании
- `touch-action: none` на canvas, чтобы система не перехватывала жесты

## Шаг 6. Что делать дальше

1. Прочитай ТЗ своего МТК — соответствующий `.docx` в корне (например, `МТК №40 техзад Библиотека. Арт.docx`).
2. Посмотри [`PLAN.md`](PLAN.md), там таблица «Карта модулей по МТК» — что для твоего МТК предполагается.
3. Если для твоего МТК **уже есть прототип** — глянь его в браузере, обсуди с пользователем направление.
4. Если **нет** — посмотри как организован МТК 38 (`mtk38-globe/`, `mtk38-poster/`, и т.д. — 8 вариантов разных метафор), сделай свой первый прототип.
5. Постоянно держи в голове `COORDINATION.md` — если нужно изменить общее правило, обсуди и зафиксируй там.

## Что точно не делать

- Не правь шрифты в `mtk38-globe/fonts/` — это shared.
- Не правь `data/ne_110m_countries.geojson` — это shared.
- Не правь файлы вариантов других МТК.
- Не редактируй секции других МТК в README/index.html.
- Не коммить напрямую в main без согласования.
- Не запускай второй HTTP-сервер на отдельном порту (мешает единой стартовой странице).
