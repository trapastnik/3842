# assets/mtk41 — справочные материалы по памятникам Ленину

Папка с собранными референсами для МТК 41 «Скульптуры». На каждый памятник из
[`data/mtk41.json`](../../data/mtk41.json) — отдельная подпапка с собранными
**фотографиями** (Wikimedia Commons + ru.wikipedia, всё под свободными
лицензиями) и **3D-моделями** (Sketchfab, ссылки).

Полную сводку см. в [CATALOG.md](CATALOG.md).

## Структура

```
assets/mtk41/
├── README.md                      ← вы здесь
├── CATALOG.md                     ← полная сводка с лицензиями
├── tools/
│   ├── _fetch2.py                 ← скачивание фото (Commons + ru.wikipedia)
│   ├── _fetch3.py                 ← top-up для редких памятников
│   └── _models.py                 ← поиск 3D-моделей (Sketchfab API)
└── <monument-id>/
    ├── photos/                    ← .jpg / .png / .JPG
    │   ├── 01_<wikimedia title>
    │   ├── 02_…
    │   └── …
    ├── photos.md                  ← каждое фото: лицензия, автор, оригинал
    └── models.md                  ← найденные 3D-модели (если есть)
```

ID памятников совпадают с `id` из `data/mtk41.json` (`alekseev-1919-bust`,
`volgograd-1973-vuchetich` и т.д.).

## Что собрано

- **46 фотографий** на 15 памятников
- **13 3D-моделей** на Sketchfab (6 памятников)
- Все фото — под CC-лицензиями или Public Domain (fair-use файлы исключены автоматически)
- На каждое фото записаны: лицензия, автор, ссылка на оригинал

См. [CATALOG.md](CATALOG.md) — таблица по каждому памятнику.

## Чего нет (и почему)

3 памятника без фото в свободных источниках:

- **Калуга, 1920-е** — старого памятника нет в Commons, нынешний в Калуге не индексирован отдельно. Возможно, найдётся в краеведческом музее.
- **Москва-Октябрьская, 1925** — установлен рабочими станции, редкий и узкоспециальный объект. Скорее всего, нужен запрос в архив РЖД или Музей московского метрополитена.
- **Бюст на «Капитале», Вознесенье, 1925** — обскурный объект в небольшом посёлке. Часть моделей кат. поселения сохранила только сам факт.

Для них в `photos.md` есть пометка о возможных архивных альтернативах (Госкаталог.рф,
краеведческие музеи).

## Что делать с 3D-моделями

Найденные модели на Sketchfab почти все **просмотровые, не для скачивания**.
Если планируется использовать в прототипе:

1. Связаться с автором (особенно с пользователем «Alex» — у него полная серия
   фотограмметрии советских памятников).
2. Или сделать собственную фотограмметрию памятников на территории музея
   (если такие есть) — DJI Mini + RealityCapture / Polycam, ~5–20 тыс. ₽ за объект.
3. Или ограничиться **стилизованными силуэтами** в прототипе без точных 3D-моделей —
   это сильно упрощает лицензионные вопросы и снижает производительные требования
   на 4K-киоске.

## Скрипты

Скрипты в `tools/` можно запустить заново (например, чтобы добавить недостающие).
Пути в них рассчитаны на работу из `assets/mtk41/tools/` — папки памятников
расположены на уровень выше:

```bash
cd assets/mtk41/tools
python3 _fetch2.py                          # все памятники с нуля
python3 _fetch2.py volgograd-1973-vuchetich # один памятник
python3 _models.py                          # все 3D-модели
```

Скрипты используют только публичные API: Wikimedia (`api.php`) и Sketchfab
(`api.sketchfab.com/v3/search`). Ничего не пишут в `data/mtk41.json` — это
отдельный источник правды.

## Использование из прототипов

Из варианта `mtk41-<имя>/` пути к ассетам — относительные через `../assets/mtk41/`:

```html
<img src="../assets/mtk41/volgograd-1973-vuchetich/photos/01_Monument...JPG">
```
