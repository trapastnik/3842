# assets/shared/fonts — централизованный шрифтовой пайплайн

Зона координатора. Модель: **централизованная кухня — у каждого МТК своя тарелка.**

## Почему не «общий склад готовых шрифтов»

Не-латинские написания нужны не только МТК 38: памятники (МТК 41) и объекты
(МТК 39) бывают по всему миру, литература (МТК 40) — на разных языках. Но **готовый
шрифт-файл шарить нельзя**: каждый МТК сжимает (subset) Noto до **только тех глифов,
что есть в его тексте**. Сабсет МТК 38 под слово «Ленин» не содержит глифов для
вьетнамской надписи на памятнике или арабского названия книги — там другие кодпоинты.

Поэтому централизуем **инструмент и исходники**, а не выходные файлы:

| Уровень | Что | Где |
|---|---|---|
| **Общее (координатор)** | пайплайн сабсеттинга + полные исходные Noto, из которых режут | `assets/shared/fonts/` (здесь) |
| **Per-МТК (владелец)** | свой сабсет `.woff2` под свой текст | `mtkXX-*/fonts/` или `mtkXX/fonts/` |

## Структура (целевая)

```
assets/shared/fonts/
├── build_fonts.py   ← общий пайплайн: на вход данные МТК → на выход сабсет .woff2
│                       (промоутится из mtk38-handoff/, после merge МТК 38)
├── noto-src/        ← полные исходные Noto Sans по письменностям (источник для сабсета)
└── README.md        ← этот файл
```

Выход (сабсет) каждый МТК кладёт **в свою зону**, не сюда.

## Как пользоваться (для сессии любого МТК)

1. В твоих данных появился не-латинский/некириллический текст (напр. название объекта
   на родном письме).
2. Запусти общий `assets/shared/fonts/build_fonts.py` против своего `data/mtkXX.json` —
   он соберёт сабсет только под нужные глифы.
3. Положи получившиеся `.woff2` в свою зону (`mtkXX-*/fonts/` или `mtkXX/fonts/`),
   подключи `@font-face`.
4. **При правке текста — перегенерируй сабсет** (иначе новых глифов в шрифте не будет).

Бренд-шрифты (Nolde, 21 Cent, 20 Kopeek) покрывают латиницу+кириллицу и остаются в
`mtk38-globe/fonts/` — их сабсетить не нужно.

## Статус (2026-06-10)

- **МТК 38 v2** уже сделал свой сабсет (29 письменностей, ~136 КБ) в `mtk38-v2/fonts/noto/`,
  скриптом `mtk38-handoff/build_fonts.py`. Self-contained, работает. Не дублируется сюда.
- **Промоут инструмента**: `build_fonts.py` от МТК 38 выносится в эту папку как общий —
  по согласованию с владельцем МТК 38 (после их merge). До этого здесь только каркас.
- Список письменностей МТК 38 — `mtk38-handoff/FONTS-NEEDED.md` (29 шт, ISO 15924).
- RTL-письменности (арабица, иврит, тхана, нко) требуют `direction: rtl` в местах вывода.
