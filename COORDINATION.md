# Координация двух Claude-сессий

В проекте параллельно работают две Claude-сессии (две git-worktree-ветки). Этот файл — общая договорённость, чтобы не наступать друг другу на ноги.

## Сессии

- **stupefied-kalam-8b2e86** (этот файл создан отсюда) — МТК 38 «Ленин на языках мира». Все визуальные варианты для одного арт-объекта.
- **bold-booth-f49a93** — МТК 39+ («Имени Ленина», «Скульптуры» и т.д.). Глобус с объектами, контент-слой.

## Файловые границы

Принцип: **каждая сессия владеет своими подкаталогами**, никто чужие файлы не правит.

### stupefied-kalam владеет
- `mtk38-globe/` — sphere-only вариант (Слои-режим удалён)
- `mtk38-poster/` — типографический постер (pretext + drift)
- `mtk38-cloud/` — 3D-облако
- `mtk38-mandala/`, `mtk38-vinyl/`, `mtk38-rain/`, `mtk38-ticker/`, `mtk38-catalog/`, `mtk38-manifest/`, `mtk38-map/` — новые варианты для МТК 38
- `index.html` (root picker), `PLAN.md`, этот `COORDINATION.md`
- Секция «МТК 38» в `README.md`

### bold-booth владеет
- `mtk39-globe/`, и в будущем `mtk40-*`, `mtk41-*`, `mtk42-*`
- `data/mtk39.json`, `data/mtk40.json`, `data/mtk41.json`, `data/mtk42.json`
- Секции «МТК 39», «МТК 40», «МТК 41», «МТК 42», «Данные» в `README.md`

### Shared, read-only для всех
- `data/ne_110m_countries.geojson` — Natural Earth (границы стран). Любая сессия может **читать**, никто не правит.
- `mtk38-globe/fonts/` — шрифты бренда. Любая сессия может **читать** (использовать через `../mtk38-globe/fonts/...`). Никто не правит. В будущем переедет в `assets/fonts/` (этап shell в [PLAN.md](PLAN.md)).

## README.md

Структура секций:
1. Введение / список содержимого
2. **МТК 38** — редактирует только stupefied-kalam
3. **МТК 39** — редактирует только bold-booth
4. **МТК 40, 41, 42** — bold-booth
5. **Данные** — bold-booth
6. **Разработка / Git** — общая, правим осторожно

Если нужно поменять что-то в общем разделе — сначала пишем в этот `COORDINATION.md`, согласуем, потом правим README.

## HTTP-сервер

**Один общий сервер на порту 8092**, запускается из корня репозитория:

```bash
cd BMK/38-42
python3 -m http.server 8092
```

Сервер отдаёт все варианты сразу:
- http://127.0.0.1:8092/ — посадочная МТК 38 (10 вариантов)
- http://127.0.0.1:8092/mtk39-globe/ — глобус МТК 39
- http://127.0.0.1:8092/data/ — общие данные

Каждая Claude-сессия видит общий результат через один URL.

Если кому-то нужен второй сервер (для изоляции при экспериментах) — используй
порт 8093 локально, но не пиши о нём в коммитах/документации без согласования.

## Git-стратегия

- Каждая сессия коммитит **только в свою feature-ветку** (`claude/stupefied-kalam-8b2e86`, `claude/bold-booth-f49a93`).
- Не коммитим напрямую в `main`.
- Merge в `main` через PR / merge commit, по одной ветке за раз. Если вторая ветка отстала — `git rebase main` или просто merge со стороны.
- Из-за файловых границ выше — конфликтов при merge почти не будет (только README, и то редко).
