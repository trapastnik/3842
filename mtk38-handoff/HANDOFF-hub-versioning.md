# Предложение координатору: версионность в общий хаб

От сессии МТК 38 (`claude/mtk38`). Дата: 2026-07-08.
Зона правок — твоя (`assets/shared/hub/`, `mtk38/index.html`, корневой `index.html`), поэтому
описываю идею и готов дать рабочий прототип. Реализация — за тобой.

## Проблема

Хаб-карусель (`assets/shared/hub/hub.js`) сейчас одноуровневая: `window.HUB_VARIANTS = [{slug,tag,name}]`
→ листаемые полноэкранные iframe. У МТК 38 появились **три поколения** решений (v1 — 8 canvas-прототипов,
v2 — арт-объект+карта на WebGL, v3 — WebGPU globe/rain/map). Сейчас v2/v3 живут отдельным лендингом
(`mtk38-v2/index.html`), в карусель `/mtk38/` не входят. Хочется держать **все версии рядом и
переключаемо**: v1 остаётся, v2 и v3 добавляются как версии.

## Идея: измерение «версия» в hub.js (обратно-совместимо)

Добавить каждому варианту необязательное поле `version`. Если оно есть хотя бы у одного варианта —
в панели появляется сегмент-переключатель версий (рядом с `Гор/Верт`), а стрелки/точки/свайп листают
**внутри активной версии**.

Схема данных (`mtkXX/index.html`):
```js
window.HUB_VARIANTS = [
  { slug: "mtk38-globe",    tag: "01 · Globe", name: "Сфера",           version: "v1" },
  { slug: "mtk38-poster",   tag: "02 · Poster",name: "Постер",          version: "v1" },
  /* …остальные 6 v1… */
  { slug: "mtk38-v2/studio",tag: "Арт-объект", name: "Ленин · арт-объект",version: "v2" },
  { slug: "mtk38-v2/map",   tag: "Карта",      name: "Карта мира",       version: "v2" },
  { slug: "mtk38-v3/globe", tag: "Глобус",     name: "Глобус (WebGPU)",  version: "v3" },
  { slug: "mtk38-v3/rain",  tag: "Дождь",      name: "Дождь",            version: "v3" },
  { slug: "mtk38-v3/map",   tag: "Карта",      name: "Карта (WebGPU)",   version: "v3" },
];
```

Изменения в `hub.js`:
1. `const versions = [...new Set(VARIANTS.map(v => v.version || '—'))];` — порядок = порядок появления.
2. Если `versions.length > 1` — отрисовать сегмент версий в баре; иначе поведение как сейчас.
3. Активная версия → `visible = VARIANTS.filter(v => (v.version||'—') === active)`; вся навигация
   (dots/goto/arrows/swipe/counter) работает по `visible`, не по всему списку.
4. Выбор версии → пересобрать точки, `goto(0)` внутри версии.
5. Deep-link: `#<slug>` по-прежнему работает — на старте находим вариант по slug, ставим его версию
   активной + индекс. (Можно и `#v3/globe`, но достаточно slug.)
6. Версия по умолчанию — предлагаю **последнюю** (новейшую) в списке; обсуждаемо.

`hub.css`: стиль сегмента версий — можно переиспользовать `.hub__orient`.

**Обратная совместимость:** у вариантов без `version` переключателя нет — другие МТК (39–42) не
затрагиваются, пока сами не добавят `version`.

## Точки входа вариантов — без нового поля

Карусель грузит `frame.src = ../<slug>/` (папка с `index.html`). Slug **может быть многосегментным**
(`mtk38-v3/globe` → `/mtk38-v3/globe/index.html`) — проверено, работает. Поэтому **`url`-поле не нужно**:
я переработаю v2/v3-решения в единый формат «папка + `index.html`» (как 8 v1-папок). Тебе останется
только логика версий, без спец-обработки путей.

(Если позже захочешь escape-hatch для внешних файлов — можно добавить `frame.src = v.url || \`../${v.slug}/\``,
но для МТК 38 не требуется.)

## Бэк-инструменты — через служебную кнопку в баре хаба (НЕ в карусели)

Бэк-инструменты МТК 38 (`mtk38-v2/back/`: `editor` — данные и приёмка, `validate`, `analysis`) —
**не витрина для посетителя**, в `HUB_VARIANTS` не идут. Но они нужны команде под рукой, поэтому —
**служебная кнопка прямо на странице МТК** (в баре хаба), а не отдельным адресом.

Механизм (симметрично `HUB_VARIANTS`): каждый `mtkXX/index.html` объявляет свои инструменты:
```js
window.HUB_TOOLS = [
  { name: "Данные и приёмка", url: "../mtk38-v2/back/editor.html" },
  { name: "Валидатор",        url: "../mtk38-v2/back/validate.html" },
  { name: "Аналитика",        url: "../mtk38-v2/back/analysis.html" },
];
```
`hub.js`: если `HUB_TOOLS` непустой — в баре появляется **дискретная служебная кнопка** («⚙ Служебное»),
по клику — маленький поповер со списком; каждый пункт открывает инструмент (новая вкладка `target=_blank`
или оверлей-iframe). Бар и так авто-скрывается через 4 c — кнопка не мозолит глаза посетителю.

**Защита от посетителя (важно — это редактор данных):** служебную кнопку по умолчанию **прятать**,
показывать только по флагу `?service=1` в URL хаба (staff открывает `…/mtk38/?service=1`) или по
скрытому жесту (напр. тройной тап по заголовку МТК). Для прод-киоска можно усилить простым PIN перед
открытием инструмента. Уровень гейта — на согласование; по умолчанию предлагаю `?service=1` + скрытый жест.

Обратная совместимость: нет `HUB_TOOLS` → кнопки нет (другие МТК не затронуты).
