## Что такое v3 (и чем отличается от v2)

v2 и v3 — **параллельные линии** одного МТК 38 «Ленин на языках мира», не замена:

| | **v2** (`claude/mtk38-worker`) | **v3** (`claude/mtk38-v3`) |
|---|---|---|
| Назначение | продакшен-демо заказчику | песочница «потолка красоты» |
| Стек | Three **r137 UMD**, WebGL | Three **WebGPU/TSL** (ESM) |
| Открытие | `file://` двойным кликом | **только http** (ESM ↛ file://) |
| Рендер | bloom/grain/виньетка 2D + WebGL-сфера | GPU-compute частицы, полный кино-пост, PBR/TSL |
| Структура | 8 композиций по v1 | те же композиции, но «тяжёлые красивые» |

Решение о сплите зафиксировано в памяти `project_mtk38_v2v3_split` (2026-06-17, под дедлайн «показать v2 заказчику»). **v3 не трогает файлы v2** — отдельная папка `mtk38-v3/`, отдельная ветка.

## Зафиксированные решения по движку (с пользователем)

- Семья **Three.js**. НЕ Babylon / НЕ PlayCanvas — второй тяжёлый движок не тащим.
- **WebGPU/TSL**: нодовые материалы (один код → WGSL+GLSL), **compute-шейдеры** для частиц/симуляций прямо на GPU. Автофоллбэк на **WebGL2** (нет WebGPU → не чёрный экран, а WebGL2-картинка).
- **R3F/drei отвергнуты** — React + JSX + бандлер против правила «vanilla JS, без бандлеров». Берём ванильный Three; кино-пост — через TSL-пост-ноды Three (и/или ванильный `postprocessing` pmndrs на WebGL2-пути — что именно, проверяется при вендоринге).
- **Все библиотеки вендорятся локально** (`mtk38-v3/vendor/`), без рантайм-CDN: офлайн-киоск = любой CDN → чёрный экран. Вендоринг разовый, при сети на dev-машине.
- Киоск = **мощный ПК + свежий Chromium** → WebGPU доступен, тяжёлый пост можно не экономить (это и есть рычаг v3).

## Архитектура: общий слой качества + рендерер под композицию

Не «один 3D-движок на все 8 вариантов»: `catalog`/`ticker`/`map` по природе 2D/типографика — тащить их в 3D-PBR против их структуры, а структурная верность v1 — выстраданное правило. Поэтому:

- **Общий слой качества (унифицируем):** пост-обработка, бренд-палитра, шрифтовой стек, `tuner.js`, данные (слова/цитаты/карточки Р2). Это и есть «новый красивый движок» в смысле единой планки.
- **Рендерер под композицию:** 3D (кольцевой глобус, cloud, rain, poster-глубина) → Three WebGPU; 2D/типографика → Canvas/DOM, **но с тем же постом и палитрой**.

```
mtk38-v3/
  vendor/         Three (ESM, WebGPU/TSL) + пост — всё локально, без CDN
  engine/
    renderer.js   WebGPURenderer + автофоллбэк WebGL2 (importmap для ESM)
    post.js       кино-пост: bloom/DOF/grain/виньетка/CA/SMAA + AgX + LUT (параметризуемый)
    text.js       4K-глифы «Ленин» (MSDF / супер-атлас canvas), 31 письменность
  tuner.js        переиспользуемая панель (слайдеры/тумблеры/сегменты/сворачивание/copy/reset)
  scenes/
    globe-rings.js  кольцевой глобус — структура v1 globe.html, но на GPU
  studio.html     полигон: globe-rings + цитаты(тумблер) + карточки Р2 (raycaster) + tuner
```

`engine/` + `tuner.js` — общий слой, который потом раскатываем на остальные композиции. `studio.html` — полигон, где утверждаем картинку, перф 4K и API тюнера.

## Как запускать (dev)

Современный Three (WebGPU/TSL) = **только ES-модули**, а ESM **не грузятся с `file://`** (CORS). Значит:

- Подними **временный** локальный http из `mtk38-v3/`, на любом порту **КРОМЕ 8092** (8092 — сервер координатора, его не трогаем):
  ```
  cd mtk38-v3 && python3 -m http.server 8093
  ```
  открыть `http://localhost:8093/studio.html`. После dev — **погасить свой сервер** (только свой).
- Прод-киоск и VPS-демо и так по http — привычка «v2 смотрим через file://» к v3-страницам не применяется.
- Headless-скрин фоллбэка: Chrome `--use-gl=angle --use-angle=swiftshader --enable-unsafe-swiftshader` по http-URL. WebGPU-картинку смотрит пользователь у себя (swiftshader WebGPU нестабилен). На macOS **нет** `timeout`; не использовать `pkill` (убьёт скриншотящий Chrome).

## Жёсткие правила (не нарушать)

- **Не пушить и не мержить в main без явной команды пользователя.** Глагол «пушь»/«мержи» — единственное разрешение.
- **Файловые границы:** зона МТК 38 — `mtk38-*/` (вкл. новый `mtk38-v3/`), `data/mtk38*.json`, `assets/mtk38/`, секция «МТК 38» в README. НЕ трогать: корневой `index.html`, `assets/shared/*` (без разрешения координатора), чужие `mtk39..42-*`, append-only `COORDINATION.md`/`PLAN.md`/`SETUP.md`/`CLAUDE.md`.
- **HTTP-сервер 8092 — только координатор.** Свой dev-сервер — другой порт, погасить после себя.
- **Только вендоренные библиотеки**, без рантайм-CDN.
- **Данные — общий источник истины** `data/mtk38.json` (52 языка, схема Р2) + `data/mtk38-quotes.json` (14 цитат). v3 их читает; канон написаний не дублирует и не переизобретает.
